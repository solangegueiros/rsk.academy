Seu primeiro smart contract terá seu nome registrado nele,
associando-o a sua wallet criada no último módulo.

Ele será utilizado para sua inscrição na RSK Academy.

Desta forma será possível identificar os estudantes :)

No futuro, também será utilizado em outros projetos do curso.

Em diversos momentos do curso utilizamos a abordagem "faça primeiro e depois entenda o que foi feito",
para que o aluno tenha o prazer de programar e entender na prática, com menos tempo exclusivo de teoria.
Esta lição acontecerá desta forma.

Utilizaremos Remix para criar seu primeiro smart contract.

Abra [remix.ethereum.org](http://remix.ethereum.org/)

Na ultima lição, já foi explicado o básico do Remix, então vamos diretamente criar um arquivo.

## Crie um smart contract

1. Clique no 2o botão do lado esquerdo - file explorer
2. Clique no primeiro símbolo à direita do browser, com a mensagem **create a new file**

![create a new file](/img/courses/dev/02/smart-contract-name/image-01.png)

Nome do arquivo / file name: **Name.sol**

![filename Name.sol](/img/courses/dev/02/smart-contract-name/image-02.png)

Copie esse exemplo:

```solidity
// SPDX-License-Identifier: MIT
pragma solidity 0.8.1;

contract Name {
    string private name;
    address public owner;

    constructor() {
        owner = msg.sender;
        name = "Your fullname";
    }

    function getName() public view returns (string memory) {
        return name;
    }
}
```

E cole aqui:

![Name.sol](/img/courses/dev/02/smart-contract-name/image-03.png)

## Name.sol

Este smart contract possui:

- A variável privada **name** para armazenar seu nome
- A variável pública **owner** para armazenar o endereço do criador do smart contract
- **constructor**, que é executado apenas no momento da publicação do smart contract
- A função **getName()** para retornar o nome armazenado na variável **name**

## Atualize seu nome

Dentro do **constructor**, na linha 10, troque a mensagem **Your fullname** pelo seu nome completo.

Por exemplo, vou trocar para _Solange Gueiros_.

```solidity
name = "Solange Gueiros";
```

Este será o resultado:

![name Solange Gueiros](/img/courses/dev/02/smart-contract-name/image-04.png)

## Interagindo com o smart contract

Quando fazemos a publicação de um smart contract utilizando Remix,
podemos encontrá-lo na parte de paixo à esquerda, no painel **Deploy and run transactions**:

![deployed contracts](/img/courses/dev/02/smart-contract-name/image-12.png)

Clique no símbolo **>** para enxergar os detalhes do smart contract **Name**:

![Deployed Name](/img/courses/dev/02/smart-contract-name/image-13.png)

Aparecerá um botão **getName**,
correspondente à a função que criamos em nosso smart contract!

E também um botão **owner**.

Mas... não criamos uma função **owner**.
Qual a origem deste botão?

Sempre que uma váriável com visibilidade pública é criada,
Solidity cria um "getter" para ela,
ou seja, uma função que retorna seu valor.

Este "getter" tem o próprio nome da variável.
Então, neste caso, ele é o botão **owner**.

Botões azuis são as funções somente leitura (read-only) e não fazem nenhuma alteração no Blockchain.
Não gastamos gás quando fazemos uma chamada para este tipo de função.

### getName

Primeiro vamos verificar o nome armazenado logo após a publicação.

Clique no botão **getName**

O resultado é o nome definido dentro do **constructor**.
No meu exemplo:

![getName](/img/courses/dev/02/smart-contract-name/image-14.png)

Na área de mensagens à direita,
podemos verificar que foi chamada a função **Name.getName()**:

![call getName](/img/courses/dev/02/smart-contract-name/image-15.png)

Clicando na linha da transação ou na seta para baixo ao lado do botão Debug,
é possível ver os detalhes da transação:

![call getName details](/img/courses/dev/02/smart-contract-name/image-16.png)

### owner

O botão owner retorna o endereço de quem fez a publicação do smart contract.
Ele foi definido dentro do **constructor**, como **msg.sender**,
que é o endereço utilizado para enviar a transação de criação do smart contract.

Clique no botão **owner**

![owner](/img/courses/dev/02/smart-contract-name/image-17.png)

Veja a lista de contas.
Você pode conferir que a conta retornada é a primeira da lista.

![first account](/img/courses/dev/02/smart-contract-name/image-20.png)

Na área de mensagens à direita,
podemos verificar que foi chamada a função **Name.owner()**:

![call owner](/img/courses/dev/02/smart-contract-name/image-18.png)

Clicando na linha da transação ou na seta para baixo ao lado do botão Debug,
é possível ver os detalhes da transação:

![call owner details](/img/courses/dev/02/smart-contract-name/image-19.png)
