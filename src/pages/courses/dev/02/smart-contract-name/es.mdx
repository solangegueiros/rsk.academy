Tu primer contrato inteligente tendrá tu nombre registrado en él,
asociándolo a tu billetera creada en el último módulo.

Esta lección sucederá de esta manera.

Usaremos Remix para crear su primer contrato inteligente.

En la última lección, ya se explicaron los conceptos básicos de Remix,
así que creemos un archivo directamente.

# Crea un smart contract

1. Haz clic en el segundo botón en el lado izquierdo - file explorer
2. Haz clic en **create a new file**

![create a new file](/img/courses/dev/02/smart-contract-name/image-01.png)

Nombre del archivo / file name: **Name.sol**

![filename Name.sol](/img/courses/dev/02/smart-contract-name/image-02.png)

Copia este ejemplo:

```solidity
// SPDX-License-Identifier: MIT
pragma solidity 0.8.1;

contract Name {
    string private name;
    address public owner;

    constructor() {
        owner = msg.sender;
        name = "Your fullname";
    }

    function getName() public view returns (string memory) {
        return name;
    }
}
```

Y pégalo aquí:

![Name.sol](/img/courses/dev/02/smart-contract-name/image-03.png)

## Name.sol

Este contrato inteligente tiene:

- Una variable privada **name** para almacenar tu nombre
- Una variable publica **owner** para almacenar la dirección del creador del contrato inteligente
- **constructor**, que se ejecuta solo en el momento de publicar el smart contract
- Una función **getName()** para devolver el nombre almacenado en la variable `name`

## Actualiza tu nombre

Dentro del **constructor**, en la línea 9, reemplace el mensaje **Your fullname** con su nombre completo.

Por ejemplo, cambiaré a _Solange Gueiros_:

```solidity
name = "Solange Gueiros";
```

Este será el resultado:

![name Solange Gueiros](/img/courses/dev/02/smart-contract-name/image-04.png)

## Interactuando con el contrato inteligente

Cuando se implementa un contrato inteligente con Remix,
podemos verlo en el panel izquierdo debajo de **Deploy and run transactions**:

![deployed contracts](/img/courses/dev/02/smart-contract-name/image-12.png)

Haz clic en el símbolo **>** para ampliar los detalles del contrato inteligente **Name**:

![Deployed Name](/img/courses/dev/02/smart-contract-name/image-13.png)

Aparecerá un botón **getName**,
correspondiente a la función que creamos en nuestro contrato inteligente!

También un botón de **owner**.

Pero ... no creamos una función de **owner**.
¿De dónde vino este botón?

Siempre que se crea una variable con visibilidad pública,
La solidez crea un "getter" para ella,
es decir, una función que devuelve su valor.

Este "getter" tiene su propio nombre de variable.
Entonces, en este caso, es el botón **owner**.

Los botones azules son funciones que son de solo lectura y no cambian nada almacenado en Blockchain.
No necesitamos gastar gasolina al usarlos.

### getName

En primer lugar, comprobaremos el nombre almacenado en la implementación.

Haz clic en el botón **getName**

El resultado es el nombre definido dentro del **constructor**.
En mi ejemplo:

![getName](/img/courses/dev/02/smart-contract-name/image-14.png)

En la parte inferior derecha,
podemos comprobar que fue una llamada a la función **Name.getName()**:

![call getName](/img/courses/dev/02/smart-contract-name/image-15.png)

Al hacer clic en la línea de la transacción o en el botón de depuración (en el lado derecho),
puedes ver más detalles de la transacción:

![call getName details](/img/courses/dev/02/smart-contract-name/image-16.png)

### owner

El botón de propietario devuelve la dirección de la persona que publicó el contrato inteligente.
Se definió dentro del **constructor**, como **msg.sender**,
que es la dirección utilizada para enviar la transacción para crear el contrato inteligente.

Haga clic en el botón **owner**

![owner](/img/courses/dev/02/smart-contract-name/image-17.png)

Consulta la lista de cuentas.
Puede verificar que la cuenta devuelta sea la primera en la lista.

![first account](/img/courses/dev/02/smart-contract-name/image-20.png)

En el área de mensajes de la derecha,
podemos ver que se ha llamado a la función **Name.owner()**:

![call owner](/img/courses/dev/02/smart-contract-name/image-18.png)

Como dijimos antes,
para ver más detalles de la transacción, puede hacer clic en la línea de la transacción o en el botón de depuración (en el lado derecho):

![call owner details](/img/courses/dev/02/smart-contract-name/image-19.png)
