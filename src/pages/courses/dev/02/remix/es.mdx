## Remix

Remix es una herramienta online. Es un IDE (Integrated Development Environment - entorno de desarrollo integrado) que se utiliza para escribir, compilar, publicar y depurar código fuente en Solidity. Se puede conectar a una billetera web y, como resultado, a cualquier red configurada en esta billetera.

Ve a
[http://remix.ethereum.org/](http://remix.ethereum.org/)

![remix.ethereum.org](/img/courses/dev/02/remix/image-01.png)

En la página de inicio / bienvenida (home / welcome), elige **environment Solidity**

![Remix environment Solidity](/img/courses/dev/02/remix/image-02.png)

## Terminal

En Remix, en la parte de abajo, a la derecha, hay una terminal con algunas bibliotecas disponibles.

Puedes enviar comandos y transacciones aquí.
Esta área también presenta el resultado de transacciones y / o llamadas a funciones de contrato inteligente.

> ¡Esta zona de retorno es muy importante para mirar los resultados!

![remix terminal](/img/courses/dev/02/remix/image-03.png)

## Compilador Solidity

En el tercer botón en el lado izquierdo, haz clic en **Solidity compiler**.

![remix solidity compiler](/img/courses/dev/02/remix/image-04.png)

Es útil **habilitar la compilación automática (auto-compile)**, para compilar contratos inteligentes automáticamente al editar en Remix.

![auto-compile](/img/courses/dev/02/remix/image-05.png)

## JavascriptVM

En Remix, en el lado izquierdo, haz clic en el botón **Deploy and run transactions** (Implementar y ejecutar transacciones).

![deploy and run transactions](/img/courses/dev/02/remix/image-10.png)

Remix tiene el ambiente **JavaScriptVM**,
un simulador de Blockchain que se ejecuta en la memoria del navegador.

![JavaScriptVM](/img/courses/dev/02/remix/image-11.png)

### Accounts / cuentas

Este simulador tiene varias direcciones / cuentas con Ethers ficticios que podemos elegir para publicar un contrato inteligente o interactuar con él.

A continuación, se muestra un ejemplo de la lista de cuentas:

![account](/img/courses/dev/02/remix/image-12.png)

En algunas versiones de Remix,
la lista de cuentas se cambia cada vez que inicia Remix o actualiza la página.

## Web3 Providers

### Web3 Injected

Web3 Injected es un proveedor web3 inyectado por un complemento de navegador como
[MetaMask](https://metamask.io/) o [Nifty](https://www.poa.network/for-users/nifty-wallet).

# Conexión de Remix con RSK testnet a través de una billetera web

Una vez que configures y selecciones la red RSK en la billetera Nifty o Metamask:

En Remix, en el panel izquierdo, haz clic en el botón **Deploy and run transactions** (Implementar y ejecutar transacciones).

![Deploy and run transactions](/img/courses/dev/02/remix/image-13.png)

En Environment, elige **Injected Web3**.

![Injected Web3](/img/courses/dev/02/remix/image-14.png)

Injected Web3 conecta Remix con la cuenta activa / seleccionada en Nifty / Metamask.

### En Metamask

Si está utilizando Metamask,
se abrirá una ventana popup para autorizar la conexión.

Primero seleccione la cuenta y haz clic en botón Next. Probablemente solo tengas una.

![Metamask select accounts](/img/courses/dev/02/remix/image-15.png)

Después haz clic en botón Connect.

![Metamask connect](/img/courses/dev/02/remix/image-16.png)

### Remix conectado

En Remix, en **Deploy and run transactions**, mira el **Custom Network**.

![Injected Web3 - ChainID 31](/img/courses/dev/02/remix/image-17.png)

**ChainID 31** se definió en la red personalizada RSK Testnet en Metamask.
